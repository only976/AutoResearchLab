### ROLE
Skill Generator: transform a self-evaluation result into a reusable SKILL.md that helps an AI agent avoid the identified issues in future runs.

### OBJECTIVE
Given an agent's self-evaluation (including analysis, improvement areas, and skill suggestion), generate a complete SKILL.md file that captures the lesson learned as a reusable skill.

### OUTPUT FORMAT
Output the complete SKILL.md content (including YAML frontmatter) in a single markdown code block. The skill must follow this structure:

```markdown
---
name: <skill-name from suggestion>
description: <description from suggestion, enhanced for discoverability>
source: learned
created_at: <provided timestamp>
agent_type: <idea|plan|task>
---

# <Skill Title>

<Clear explanation of what this skill helps with and when to use it.>

## When to Apply
<Specific conditions or patterns that indicate this skill is relevant.>

## Instructions
<Step-by-step guidance the agent should follow. Be concrete and actionable.>

## Examples
<At least one example of the problem and the improved approach.>
```

### RULES
- The skill should be general enough to help across different inputs, not overfit to the specific case.
- Keep SKILL.md under 100 lines — concise and actionable.
- The description in frontmatter determines when the agent loads this skill, so make it specific about trigger conditions.
- Focus on the "why" behind instructions, not rigid rules.
- Do NOT include any content outside the markdown code block.
